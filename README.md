# event-manager-service
Web Api для сервиса управления мероприятиями

## Запуск

### Для сборки решения:
1. Перейдите в корневую папку решения
2. Запустите *dotnet build*

### Для Запуска тестов:
1. Из корневой папки перейдите в папку тестового проекта  cd .\EventService.Tests\
2. Выполните *dotnet test*

### Для запуска проекта 
1. Из корневой папки решения перейдите в папку проекта  cd .\EventManagerService\
2. Используйте *dotnet run -lp http* (по умолчанию используется порт 5214) или *dotnet run*
3. После успешного запуска откройте http://localhost:port/swagger/index.html для доступа к swagger, вместо port подставьте порт из вывода dotnet run

## Описание API:

### 1. GET /events

#### query параметры
    1. *title* - *string*
    Фильтр для поиска по наименованию мероприятия. Поиск осуществляется по частичному совпадению, регистронезависимый.
    2. *from* - *DateTime*
    Фильтр для начальной даты мероприятия. Находит все мероприятия, начинающиеся не позже данной даты
    3. *to* - *DateTime*
    Фильтр для конечной даты мероприятия. Находит все мероприятия, которые завершаются не позже данной даты
    4. *page* - *int*
    Номер страницы. Принимает значения от 1 до *int.MaxValue*. Значение по умолчанию - 1
    5. *pageSize*
    Количество мероприятий на странице. Принимает значения от 10 до 100. Значение по умолчанию 100.

#### Формат ответа:

##### Модель ответа содержит:
  Массив *events*, содержащий все события, подходящие под заданные фильтры и расположенные на указанной странице
  Общее количество найденных событий по заданным фильтрам *total*
  Номер текущей страницы *page*
  Размер текущей страницы *currentPageSize*

```{
  "events": [
    {
      "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
      "title": "string",
      "description": "string",
      "startAt": "2026-04-07T07:51:10.309Z",
      "endAt": "2026-04-07T07:51:10.309Z"
    }
  ],
  "total": 0,
  "page": 0,
  "currentPageSize": 0
}```

### Формат ошибок

Для отправки информации об ошибках используется стандартный формат Problem Details

*Type*	
Ссылка на URI [RFC3986], идентифицирующая тип проблемы.
*Title*	
Краткая, удобочитаемая сводка по типу проблемы.
*Status*	
Код состояния HTTP, созданный сервером-источником для этого возникновения проблемы.
*Instance*	
Ссылка на URI, идентифицирующая конкретное возникновение проблемы
